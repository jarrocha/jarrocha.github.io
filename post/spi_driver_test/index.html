STM32 SPI Driver Testing with Logic Analyzer

Posted by Jaime Arrocha on Sunday, September 19, 2021

TOC

Intro

This is a short post on how to use a Logic Analyzer to test an SPI driver implementation when not having another device to test with.

I recently bought a Saleae Logic Analyzer and it comes with a very convenient protocol analyzer tool to test SPI, I2C, CAN and other protocols. Here is the web guide.

I wanted to test this for myself so I updated and uploaded an old SPI driver I wrote for STM32F446. Given that it has been a while since I played around with this protocol, I thought it could be a good time to practice the basics. Here is the link. Not all of the code will compile since I have been gradually updating it, but this one does.

Implementation

Here’s the test code. It configued the STM32 as master and enabled only the MOSI and SCLK pins for SPI communication. The details on how to interact with the driver are not difficult to deduce since I’m using basic configuration for GPIO ports and SPI.

 1/*
 2 * 006_SPI_SendTest.c
 3 *
 4 */
 5
 6/** Setup
 7 * SPI2_NSS  --> PB12 AF5
 8 * SPI2_SCLK --> PB13 AF5
 9 * SPI2_MISO --> PB14 AF5
10 * SPI2_MOSI --> PB15 AF5
11 */
12
13#include <string.h>
14#include "stm32f446xx_gpio_driver.h"
15#include "stm32f446xx_spi_driver.h"
16
17void GPIOInit()
18{
19	// Configure MOSI Pin
20	GPIO_Handle_t spiPin;
21	memset(&spiPin, 0, sizeof(spiPin));
22
23	// Both pins share similar configurations
24	spiPin.pGPIOx = GPIOB;
25	spiPin.pGPIOx_pin.GPIO_PinMode = GPIO_MODE_ALTFN;
26	spiPin.pGPIOx_pin.GPIO_PinAltFunMode = BIT5;
27
28	// Setting up MOSI pin
29	spiPin.pGPIOx_pin.GPIO_PinNumber = BIT15;
30	GPIO_Init(&spiPin);
31
32	// Setting up SCLK pin
33	spiPin.pGPIOx_pin.GPIO_PinNumber = BIT13;
34	GPIO_Init(&spiPin);
35}
36
37void SPIInit()
38{
39	// Configure spi2 for master communication
40	SPI_Handle_t spi2;
41	memset(&spi2, 0, sizeof(spi2));
42
43	spi2.pSPIx = SPI2;
44	spi2.SPIConfig.SPI_BusConfig = SPI_BUS_CFG_FULLD;
45	spi2.SPIConfig.SPI_DeviceMode = SPI_DEVICE_MODE_MASTER;
46	spi2.SPIConfig.SPI_Speed = SPI_CLK_SPEED_DIV2;
47	spi2.SPIConfig.SPI_DFF = SPI_DFF_MODE_8BIT;
48	spi2.SPIConfig.SPI_CPHA = SPI_CLKPHA_LOW;
49	spi2.SPIConfig.SPI_CPOL = SPI_CLKPOL_LOW;
50	spi2.SPIConfig.SPI_SSM = SPI_SSM_ENABLE;
51	SPI_Init(&spi2);
52
53	// Manually setting SSI to high to avoid modf error.
54	SPI_SSIConfig(spi2.pSPIx, ENABLE);
55
56	// Enable device after configuration
57    SPI_SetupDevice(spi2.pSPIx, ENABLE);
58}
59
60int main()
61{
62	GPIOInit();
63
64	SPIInit();
65
66	// Send data
67	char buff[] = "Hello World";
68	SPI_SendData(SPI2, (uint8_t*)buff, strlen(buff));
69
70	while (1);
71
72	return 0;
73}

You can see that for the GPIO ports are setup for using Alternate Functioning (AF) (You can refer to the reference manual here) and we are simply configuring the AF mode 5 and the pin numbers 15 for MOSI and 13 for SCLK. The driver does the peripheral enabling and register configuration. After configuration we are simply sending a “Hello World” message through the pins to be read by the logic analyzer.

Here’s a quick example on how to choose ports for SPI on this board. You can see from the picture below the last three rows belong to the chosen ports for MOSI (PB15) and SCLK (PB13) configured for AF5.

Here’s the pin map for the board. I’ve chosen again SCLK (PB13) and MOSI (PB15) . This will be connected to Channel 0 and Channel 1 respectively to the logic analyzer.

Here’s a picture of how they are connected. The up most black cable is connected to ground on header pin 9 (refer to pin map picture above), the red cable can be ignored.

Results

The results are interesting. After just a couple of tweaks on the logic analyzer, you will be able to read the message. This was tested on a Ubuntu machine.
